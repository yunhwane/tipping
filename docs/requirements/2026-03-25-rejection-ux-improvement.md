# 반려 UI/UX 개선

## 배경

검수 시스템 리팩토링(PR #4) 이후 반려 기능은 동작하지만, 작성자 입장에서 반려 사유 확인과 수정 재제출 흐름이 직관적이지 않음. 알림에서도 승인/반려 구분이 없고, 카드에서도 반려 상태가 눈에 잘 띄지 않음.

## 요구사항

### 1. 상세 페이지 반려 배너 개선
- 현행 한 줄 빨간 배너 → 알럿 스타일 배너로 변경
- 그래디언트 배경 + 경고 아이콘 + "이 글은 반려되었습니다" 헤딩
- 반려 사유 텍스트 + 검수 일시 표시
- 오른쪽에 "수정하기 →" 인라인 CTA 버튼 (작성자에게만)
- Tip, Project 상세 페이지 모두 적용

### 2. 프로필 카드 반려 표시 개선
- 반려된 카드: 상단 컬러 바 빨간색, 보더 빨간색으로 전체 강조
- 반려 사유를 별도 박스(bg-red-50)에 표시
- "수정하기" CTA 버튼 추가
- TipCard, ProjectShowcaseCard 모두 적용

### 3. 알림 메시지 개선
- 반려 알림: 빨간 X 아이콘 + 왼쪽 빨간 보더 + 빨간 배경
- 승인 알림: 초록 체크 아이콘
- 반려 사유를 별도 박스에 분리 표시
- "수정하기 →" CTA 포함
- 알림 텍스트: `"제목" 이(가) 반려되었습니다.` 형식

### 4. 백엔드 변경
- 알림 메시지 포맷 변경 (사유는 message에서 분리, 제목만 포함)
- Notification 모델의 type 필드 활용하여 프론트에서 승인/반려 구분

## 범위

### 변경 대상 파일
| 파일 | 변경 |
|------|------|
| `src/app/tips/[id]/page.tsx` | 반려 배너 → 알럿 스타일 + CTA |
| `src/app/projects/[id]/page.tsx` | 동일 |
| `src/components/tip-card.tsx` | 반려 시 카드 전체 강조 + CTA |
| `src/components/project-showcase-card.tsx` | 동일 |
| `src/components/notification-bell.tsx` | 타입별 아이콘/색상 + 사유 박스 + CTA |
| `src/server/api/helpers/content-review.ts` | 알림 메시지 포맷 변경 |

### 범위 외
- 관리자 반려 사유 입력 방식 (현행 자유 입력 유지)
- 이메일 알림
- 새 DB 모델/필드 추가
