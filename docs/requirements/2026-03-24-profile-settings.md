# 프로필 설정 기능

**작성일**: 2026-03-24
**브랜치**: `feat/profile-settings`
**상태**: 검토 중

---

## 1. 배경

현재 `/profile` 페이지는 사용자 정보 조회와 콘텐츠(팁, 북마크, 프로젝트) 탭만 존재한다.
사용자가 프로필 사진, 닉네임, 비밀번호를 직접 변경할 수 있는 설정 기능이 필요하다.

## 2. 요구사항

### 2.1 프로필 사진 변경 (Mock)
- 프로필 사진 클릭 시 변경 UI 표시
- **현재 단계**: Mock 구현 (미리 정의된 아바타 목록에서 선택)
- **향후 계획**: 외부 서비스(S3/Cloudinary) 연동으로 파일 업로드 지원
- 선택한 아바타 URL을 User.image에 저장

### 2.2 닉네임 변경
- 현재 닉네임 표시 + 수정 가능한 입력 필드
- 유효성 검증:
  - 필수 입력
  - 2~20자 제한
  - 앞뒤 공백 제거(trim)
- 변경 시 세션 정보 업데이트 반영

### 2.3 비밀번호 변경
- 현재 비밀번호 확인 → 새 비밀번호 입력 → 새 비밀번호 확인
- 유효성 검증:
  - 현재 비밀번호 일치 확인 (bcrypt compare)
  - 새 비밀번호: 8자 이상, 영문 + 숫자 포함 (기존 회원가입 규칙과 동일)
  - 새 비밀번호 확인 일치
- 성공 시 토스트/알림 표시

## 3. 범위

### 포함
- `/profile` 페이지에 "설정" 탭 추가
- `user` tRPC 라우터 생성 (프로필 조회/수정, 비밀번호 변경)
- 설정 탭 UI 컴포넌트

### 제외
- 실제 파일 업로드 (외부 스토리지 연동은 향후 별도 작업)
- 이메일 변경 (이메일 인증 재발송 로직 필요, 별도 작업)
- 계정 삭제

## 4. 기술 접근

### 4.1 백엔드 (tRPC Router)

**새 라우터**: `src/server/api/routers/user.ts`

| Procedure | Type | Auth | 설명 |
|-----------|------|------|------|
| `getProfile` | query | protected | 현재 사용자 프로필 조회 |
| `updateProfile` | mutation | protected | 닉네임, 이미지 URL 변경 |
| `changePassword` | mutation | protected | 비밀번호 변경 |

### 4.2 프론트엔드

**설정 탭 UI 구조**:
```
설정 탭
├── 프로필 섹션
│   ├── 아바타 선택 (Mock: 미리 정의된 아바타 그리드)
│   ├── 닉네임 입력 필드
│   └── [저장] 버튼
│
└── 비밀번호 변경 섹션
    ├── 현재 비밀번호 입력
    ├── 새 비밀번호 입력
    ├── 새 비밀번호 확인 입력
    └── [비밀번호 변경] 버튼
```

### 4.3 Mock 아바타
- DiceBear 또는 유사 API를 활용한 미리 생성된 아바타 URL 목록
- 예: `https://api.dicebear.com/9.x/avataaars/svg?seed=1` ~ `seed=12`
- 사용자가 그리드에서 클릭하여 선택

## 6. Mock → 실제 전환 계획

프로필 사진 기능은 인터페이스를 분리하여 설계:
1. **현재 (Mock)**: 미리 정의된 아바타 URL 선택
2. **향후 (실제)**: 파일 업로드 컴포넌트 + 외부 스토리지 연동
   - `updateProfile`의 `image` 필드는 동일하게 URL 문자열을 받으므로 백엔드 변경 없음
   - 프론트엔드 아바타 선택 UI만 파일 업로드 UI로 교체
